إنهاء الجلسة وتحديث كل ملفات الحوكمة.

نفّذ بالترتيب بالضبط:

## الخطوة 1: حدّث SESSION_HANDOFF.md
اكتب/حدّث الملف بالهيكل التالي:

```markdown
# Session Handoff — [تاريخ اليوم بصيغة YYYY-MM-DD]

## ما تم إنجازه اليوم
- [نقطة واحدة لكل إنجاز — مع مسار الملف المتأثر]

## ما لم يكتمل
- [كل نقطة + السبب]

## القرارات الجديدة
- [كل قرار + مرجع DECISIONS_LOG]

## الملفات المتأثرة
- apps/api/src/... (مع وصف التغيير)

## الاختبارات المنفذة
- [ما شغلناه من tests + النتيجة: ✅/❌]

## المخاطر المفتوحة
- [كل خطر جديد]

## ممنوع تغييره في الجلسة القادمة
- [قرارات مقفلة اليوم]

## الخطوة التالية بالضبط
- [تعليمات دقيقة وواضحة — لما يفتح الجلسة الجديدة يبدأ من هنا]
```

## الخطوة 2: حدّث MODULE_STATUS_BOARD.md
غيّر حالة كل وحدة اشتغلنا عليها:
- `░░░░░░░░░░` = لم يبدأ
- `████░░░░░░` = قيد التنفيذ
- `████████░░` = جاهز للاختبار
- `██████████` = مكتمل ومجتاز

## الخطوة 3: حدّث DECISIONS_LOG.md
أضف أي قرار معماري جديد اتخذناه:
```markdown
| DX | القرار | البديل المرفوض | السبب | YYYY-MM-DD |
```

## الخطوة 4: حدّث OPEN_ISSUES.md
أضف أي مشكلة جديدة — أضف حالة: 🔴 حرج / 🟡 متوسط / 🟢 بسيط

## الخطوة 5: تحقق من الكود
شغّل:
```bash
npm run build    # في apps/api
npm run test     # في apps/api
```
إذا فيه أخطاء — أصلحها قبل ما تكمل.
