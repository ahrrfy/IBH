خطّط للجلسة الحالية — حدد المهمة وافهم النطاق قبل البدء.

قبل ما تبني أي شي، نفّذ:

## 1. اقرأ الحالة الحالية
- `governance/SESSION_HANDOFF.md` ← الخطوة التالية
- `governance/MODULE_STATUS_BOARD.md` ← الوضع الحالي
- `governance/OPEN_ISSUES.md` ← مشاكل مفتوحة

## 2. حدد المهمة بدقة

اسأل نفسك (واطبع الإجابات):

```
📋 خطة الجلسة
═══════════════════════════════════════
🎯 الهدف:       [ماذا نبني بالضبط؟]
📍 البداية:     [من أي ملف/وحدة نبدأ؟]
📦 المخرجات:    [ما الملفات اللي لازم تنتج؟]
🧪 الاختبار:    [كيف نتحقق إنها شغالة؟]
⚡ المخاطر:     [ما الذي ممكن يتعطل؟]
📏 النطاق:      [ما داخل النطاق — ما خارجه]
⏱️ التقدير:    [كم جزء من الجلسة يحتاج؟]
═══════════════════════════════════════
```

## 3. راجع التبعيات

قبل ما تبدأ، تأكد:
- [ ] الوحدات اللي أعتمد عليها مكتملة (من MODULE_STATUS_BOARD)
- [ ] ما يوجد conflict مع OPEN_ISSUES
- [ ] ما أحتاج قرار جديد — أو أضيفه لـ DECISIONS_LOG

## 4. اعتمد على صاحب المشروع

اطبع الخطة واسأل:
"هل هذه هي المهمة المطلوبة؟ أم فيه تعديل؟"

> ⚠️ لا تبدأ الكتابة قبل ما تحصل موافقة صاحب المشروع على الخطة
