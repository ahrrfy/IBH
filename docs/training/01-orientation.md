# جلسة التدريب 1 — أساسيات النظام والتنقل

> **الجمهور:** كل المستخدمين الجدد · **المدة:** 90 دقيقة (60 تدريب + 30 تطبيق)
> **المتطلبات المسبقة:** حساب فعّال + متصفح Chrome/Firefox محدّث.

---

## الأهداف

بنهاية الجلسة، يستطيع المتدرب:
1. تسجيل الدخول وفهم تنبيهات 2FA.
2. التنقل عبر Activity Bar + Sub‑Sidebar.
3. تفسير شارات الحالة (مسودة / معتمد / مغلق).
4. معرفة "ماذا أستطيع رؤيته" مقابل "ما لا أستطيع رؤيته" حسب الدور.
5. الإبلاغ عن خطأ بشكل صحيح (تذكرة + لقطة شاشة).

---

## المحتوى

### 1. تسجيل الدخول و2FA (10 دقائق)
- صفحة `/login` — البريد + كلمة المرور.
- كلمة مرور للمرة الأولى → فرض تغييرها.
- تفعيل 2FA من `/settings/profile/2fa` — رمز Authenticator + 8 أكواد احتياطية.
- ⚠️ كل محاولة دخول تُسجَّل في `audit_logs` مع IP و user-agent.

### 2. الـ Activity Bar (10 دقائق)
الشريط العمودي يميناً — أيقونات الوحدات الرئيسية المرئية للدور.
- **مدير النظام:** يرى كل الوحدات.
- **محاسب:** Finance + Reports + Sales + Purchases + Assets + Inventory.
- **كاشير:** POS + CRM + Inventory.
- (راجع `permissions.ts` للقائمة الكاملة لكل دور.)

### 3. Sub‑Sidebar (10 دقائق)
عند تحديد وحدة، يظهر Sub‑Sidebar على الشمال يعرض أقسام الوحدة.
- مثال Sales: Invoices · Orders · Quotations · Customers · Returns.

### 4. شارات الحالة (10 دقائق)
- 🟢 active / posted / approved
- 🟡 draft / in_progress / soft_closed
- 🔴 rejected / cancelled / hard_closed

### 5. الصلاحيات (15 دقائق)
- 7 مستويات لكل وحدة: C/R/U/D/S/A/P.
- مفهوم Field‑Level Security (مثلاً Branch Manager لا يرى مبالغ > 5M).
- لو ظهر 403، الإبلاغ لمدير النظام (ليس الخطأ).

### 6. تطبيق عملي (30 دقيقة)
- كل متدرب: تسجيل دخول → تفعيل 2FA → فتح Sales → إصدار quote تجريبية.
- مدرّب يلاحظ ويعلّق.

---

## ملاحظات للمدرّب

- اسأل كل متدرب عن دوره الفعلي قبل البدء — Sub‑Sidebar الذي يراه يختلف.
- لو وجد مستخدم لا يرى وحدة يحتاجها فعلاً → سجّل ذلك في تذكرة "تعديل الدور" بعد الجلسة.
- ركز على **"النظام يفكر، الموظف ينقر"** — إذا اضطر المتدرب لحفظ خطوات معقدة، فهذا عيب يستحق الإبلاغ عنه.
