# جلسة التدريب 2 — المبيعات و POS

> **الجمهور:** الكاشير + مدير المبيعات · **المدة:** 90 دقيقة.
> **المتطلبات:** حساب بدور `Cashier` أو `SalesManager` + جهاز POS مُسجَّل.

---

## الأهداف

1. فتح وردية وإغلاقها بشكل صحيح.
2. إصدار فاتورة بيع نقدية وآجلة.
3. استرجاع منتج (مرتجع) مع توليد قيد عكسي.
4. تحويل عرض سعر → طلب → فاتورة دون إعادة إدخال.

---

## المحتوى

### 1. فتح الوردية (10 دقائق)
- `/pos/shifts/new` → اختر الجهاز + رصيد افتتاحي → افتح.
- لا يمكن فتح وردية ثانية على نفس الجهاز قبل إغلاق السابقة (Partial Unique Index — `shifts_one_open_per_device`).

### 2. إصدار إيصال POS (15 دقائق)
- شاشة POS → بحث منتج (باركود/اسم) → كميّة → دفع.
- طرق الدفع: نقد، بطاقة، محفظة موبايل.
- `clientUlid` يُرسَل لضمان idempotency — لو انقطع الإنترنت وتمت إعادة الإرسال، الإيصال يُسجَّل مرة واحدة.

### 3. فاتورة بيع آجلة (10 دقائق)
- `/sales/invoices/new` → اختر عميل (تأجيل) → بنود → "حفظ كمسودة".
- "إصدار" يولّد القيد المحاسبي (Mr 221 / Mt 511 + 411 ضريبة).

### 4. تحويل العروض → الطلبات → الفواتير (15 دقائق)
- `/sales/quotations/new` → عرض → "تحويل لطلب" → طلب → "تحويل لفاتورة".
- البنود تنتقل تلقائياً، لا حاجة لإعادة الإدخال.

### 5. المرتجعات (15 دقائق)
- `/sales/returns/new` → اختر فاتورة المصدر → الكميّة المرتجعة لكل بند.
- عند الاعتماد: قيد عكسي + إعادة المخزون + إيصال ائتمان للعميل.

### 6. إغلاق الوردية (10 دقائق)
- `/pos/shifts/<id>/close` → عدّ النقدية الفعلي → النظام يقارن مع المتوقع.
- فرق > 5,000 IQD يحتاج موافقة مدير المبيعات.

### 7. تطبيق عملي (15 دقيقة)
- كل متدرب: 3 إيصالات + 1 فاتورة آجلة + 1 مرتجع + إغلاق وردية.

---

## نقاط حذر

- **لا تحذف فاتورة معتمدة** — استخدم زر "عكس/إلغاء" → يولّد قيداً عكسياً.
- **لا تُغيّر سعر بنود فاتورة معتمدة** — أنشئ مرتجعاً + فاتورة جديدة.
- لو فشل الدفع بالبطاقة في POS → اضغط "إلغاء" قبل خروج العميل، النظام يحرر القفل على الإيصال.
