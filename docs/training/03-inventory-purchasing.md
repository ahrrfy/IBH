# جلسة التدريب 3 — المخزون والمشتريات

> **الجمهور:** أمين المستودع + مسؤول المشتريات · **المدة:** 90 دقيقة.

---

## الأهداف

1. إنشاء أمر شراء واستلام البضاعة (GRN) ومطابقتها بفاتورة المورد (3‑Way Match).
2. تنفيذ تحويل بين مستودعين.
3. إجراء جلسة جرد وإصدار قيد التسوية.
4. فهم Moving Weighted Average وأثر كل حركة على التكلفة.

---

## المحتوى

### 1. إنشاء أمر شراء (10 دقائق)
- `/purchases/orders/new` → مورد + مستودع وجهة + بنود + شروط دفع.
- بعد الاعتماد، الحالة `approved` ولا يمكن تعديل البنود.

### 2. استلام البضاعة GRN (15 دقائق)
- `/purchases/grn/new` → اختر PO → كمية مستلمة فعلية لكل بند.
- "اعتماد فحص الجودة" → يضيف للمخزون + ينشئ قيد GR/IR.
- إذا الكمية < المطلوب: PO يبقى مفتوحاً جزئياً.

### 3. فاتورة المورد و3‑Way Match (15 دقائق)
- `/purchases/invoices/new` → اختر PO + GRN → النظام يطابق:
  - PO.qty = GRN.qty = Invoice.qty (بحدود tolerance ± 2%).
  - PO.price = Invoice.price (بحدود ± 5%).
- الفروقات تُسجَّل في حساب الفروقات قبل ترحيل الفاتورة.

### 4. تحويلات المخزون (10 دقائق)
- `/inventory/transfers/new` → من + إلى + بنود → "بدء التحويل".
- "اعتماد" → ledger entries OUT من المصدر و IN للوجهة + تكلفة موزونة.

### 5. الجرد (15 دقائق)
- `/inventory/stocktaking/new` → مستودع → "بدء جلسة".
- إدخال الكميات الفعلية → النظام يحسب الفروق + قيمتها (MWA).
- "اعتماد + ترحيل" → قيد تسوية لـ 611 (COGS) أو 593 (إيرادات متنوعة) حسب الفرق.

### 6. Moving Weighted Average (10 دقائق)
- شرح المعادلة: `(qty_old × cost_old + qty_in × cost_in) / (qty_old + qty_in)`.
- مثال عملي: 100 وحدة @1,000 + 50 وحدة @1,200 → MWA = 1,067 IQD.
- StockLedger مع append‑only — لا تعديل تاريخي للتكلفة.

### 7. تطبيق عملي (15 دقائق)
- مهمة كاملة: PO → GRN → Invoice → Transfer لمستودع آخر → Stocktaking.

---

## نقاط حذر

- **لا تستلم بضاعة بدون PO** (إلا في حالات خاصة بصلاحية مدير المخزن).
- **لا تعدّل StockLedger يدوياً** — كل حركة عبر مستند مصدر (`ref_type` + `ref_id`).
- **رصيد سالب ممنوع** — الـ DB يرفض، إلا بصلاحية خاصة `Inventory:negative_stock_override`.
