# جلسة التدريب 4 — المالية والإغلاق الشهري

> **الجمهور:** المحاسب + المدير المالي · **المدة:** 90 دقيقة.

---

## الأهداف

1. فهم Double‑Entry وكيف يولِّد النظام القيود تلقائياً.
2. تنفيذ مطابقة بنكية كاملة.
3. إغلاق شهري عبر Wizard الـ 7 خطوات.
4. قراءة قائمة الدخل والمركز المالي وميزان المراجعة.

---

## المحتوى

### 1. Double‑Entry والـ Posting Engine (10 دقائق)
- كل عملية تجارية = قيد محاسبي تلقائي.
- DB CHECK: `total_debit = total_credit` — لا يمكن حفظ قيد غير متوازن.
- Hash chain على `journal_entry` — أي تلاعب يكشفه `verifyChain()`.
- **لا تعديل بعد الترحيل** — عكس فقط بقيد عكسي.

### 2. دليل الحسابات (10 دقائق)
- `/finance/chart-of-accounts` — شجرة هيكلية.
- 6 تصنيفات: أصول ثابتة / متداولة / خصوم / حقوق ملكية / إيرادات / مصروفات.
- `category` و `accountType` غير قابلَين للتغيير بعد استخدام الحساب في قيود (T06 guard).

### 3. القيود اليومية (10 دقائق)
- `/finance/journal-entries` — كل قيد له `referenceType` + `referenceId` (مصدر العملية).
- معظم القيود يتم توليدها تلقائياً عبر Posting Engine.
- القيود اليدوية (إن لزمت) تحتاج صلاحية `GL:Submit` ومراجعة `GL:Approve`.

### 4. المطابقة البنكية (20 دقيقة)
- `/finance/banks` → اختر الحساب → "مطابقة".
- "بدء مطابقة جديدة" → تاريخ كشف + رصيد كشف.
- النظام يحسب `bookBalance` من القيود الموجودة قبل تاريخ الكشف.
- لكل بند: "تطابق" أو "فك" — الفرق `discrepancy` يُحدَّث مباشرة.
- "إضافة تسوية" لرسوم بنكية أو فوائد لم تظهر في القيود.
- "إكمال المطابقة" مفعَّل فقط حين `discrepancy = 0` (T16).

### 5. الإغلاق الشهري — 7 خطوات (25 دقيقة)
1. **ترحيل القيود** — التأكد من عدم وجود مسودات في الفترة.
2. **مطابقة البنوك** — كل الحسابات النشطة مطابقة لتاريخ نهاية الفترة.
3. **تقييم المخزون** — تأكيد قيمة المخزون من تقرير `/inventory/valuation`.
4. **تسويات نهاية الشهر** — إهلاكات + استحقاقات + مخصصات.
5. **إقفال مبدئي (Soft Close)** — يمكن إعادة الفتح خلال 30 يوماً (مدير النظام فقط).
6. **لقطة القوائم المالية** — حفظ نسخة P&L + الميزانية كـ snapshot.
7. **إقفال نهائي (Hard Close)** — لا تراجع، الفترة تصبح read‑only لكل الأدوار.

> جدول الـ Wizard في `/finance/periods/<id>/close` يفرض الترتيب — لا يمكن القفز فوق خطوة (T17).

### 6. القوائم المالية (10 دقائق)
- `/finance/trial-balance` — ميزان المراجعة (المدين = الدائن).
- `/finance/income-statement` — قائمة الدخل بفترة.
- `/finance/balance-sheet` — المركز المالي بتاريخ.
- `/finance/cash-flow` — التدفقات النقدية.

### 7. تطبيق عملي (15 دقيقة)
- مطابقة بنكية كاملة لحساب اختباري.
- محاكاة 7 خطوات إغلاق على فترة وهمية.

---

## نقاط حذر

- **لا تكتب في فترة مغلقة** — حتى بصلاحية `super_admin` لو الفترة hard_closed.
- **عكس قيد** ≠ حذف — يبقى الأصل + يضاف عكسي يربطهما `referenceId`.
- **DB Trigger** يمنع UPDATE/DELETE على `journal_entry_lines` — أي محاولة تفشل بصمت محاسبي.
